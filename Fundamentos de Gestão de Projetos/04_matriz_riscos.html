Identificação, Classificação e Estratégias de Mitigação
# Risco Prob. Impacto Score Estratégia de Mitigação Responsável
R1 Alucinações do LLM Insights incorretos ou fabricados entregues como fato ao time de negócio 4 5 20 Human-in-the-loop obrigatório; fontes visíveis por insight; circuit breaker para outputs com baixa confiança; testes com dados reais antes do launch Tech Lead + PM
R2 Vazamento de dados (PII) Dados sensíveis de consumidores expostos via logs, prompts ou respostas da API de LLM 3 5 15 Anonimização/pseudonimização obrigatória pré-LLM; DPA formal com provedor; auditoria de logs; política de retenção de dados Eng. Segurança + DPO
R3 Baixa adoção Usuários não confiam nos insights gerados ou acham o produto difícil de usar 3 4 12 Usuários reais no processo de design desde Fase 0; modo "debug" mostrando tickets-fonte de cada insight; onboarding guiado PM + UX Designer
R4 Dependência de provedor LLM Indisponibilidade, mudança de preço ou descontinuação da API impacta a operação 3 4 12 Arquitetura multi-provider (OpenAI + Claude + fallback local); caching de respostas; monitoramento de custo por request Arquiteto de Solução
R5 Escopo creep Solicitações de novas funcionalidades por stakeholders diluem o foco e atrasam o MVP 4 3 12 Backlog público e priorizado; revisão quinzenal de escopo; PM com autoridade formal para recusar itens fora do roadmap Product Manager
R6 Qualidade dos dados de entrada Tickets mal formatados, duplicados ou incompletos prejudicam a qualidade dos insights 4 3 12 Pipeline de pré-processamento robusto (normalização, deduplicação, validação); relatório de qualidade exibido no momento da importação Eng. de Dados
R7 Conformidade LGPD / GDPR / EU AI Act Dados de atendimento com PII sujeitos a regulação — não conformidade pode inviabilizar a operação 2 5 10 Mapeamento de dados pré-MVP; T&C revisados por advogado; suporte a right to be forgotten; monitoramento do EU AI Act para operações na UE Jurídico + DPO
R8 Risco de mercado novo Concorrentes estabelecidos (Zendesk, Salesforce) embarcando IA nativa reduzem o espaço de mercado 3 3 9 Posicionamento mid-market; estratégia de integração (complementar, não concorrente); diferenciação via efeito de rede dos dados acumulados CEO + PM
Metodologia: Framework PMI/PMBOK · Score = Probabilidade (1–5) × Impacto (1–5)
Estratégias: Evitar · Mitigar · Transferir · Aceitar
